Overview

What is this project?

TestApp is a REST API for managing tasks and projects. Think of it like Trello's backend — it handles creating, assigning, and tracking tasks through a clean JSON API.

Audience: Development teams who need a lightweight, self-hosted task management backend.

Approach: A Node.js/Express server connects to PostgreSQL via Prisma ORM, providing RESTful endpoints with JWT authentication.

Entry Points

Where does it start?

src/index.ts
Trigger: npm start

When you run npm start, this file runs first. It calls loadConfig() to read environment variables, connectDB() to open the PostgreSQL pool, then creates the Express app. All middleware is registered, routes are mounted, and app.listen(3000) starts the server.

Modules

Key files explained

Auth Service
src/services/auth.ts
service

This handles proving who a user is. When someone logs in, it checks their email and password against the database. If correct, it creates a JWT token for future requests.

Core functions: login(email, password), refreshToken(token), logout(userId). Uses bcrypt for password comparison and stores refresh tokens in Redis with 30-day TTL.

If Redis is down, refresh token validation fails even for valid tokens.

Glossary

Terminology

JWT
JSON Web Token. A signed string that proves who a user is without hitting the database on every request. Generated on login, sent in the Authorization header for subsequent requests.
Middleware
A function that runs on every request before it reaches the route handler. Used here for authentication checks and request logging.
ORM
Object-Relational Mapper. Prisma lets you write TypeScript code that automatically translates to SQL queries, so you don't write raw SQL.
